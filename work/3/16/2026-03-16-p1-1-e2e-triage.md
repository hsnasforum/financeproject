## 변경 이유
- `analysis_docs/v2` 상태판에서 `[보류]`가 과도하게 넓게 적용돼 실제 진행 상태와 어긋나 있었습니다.
- `P1-1` closeout gate를 막는 `pnpm e2e:rc` 실패가 Public IA 변경의 직접 회귀인지, 기존 selector/copy drift인지 먼저 분리할 필요가 있었습니다.

## 핵심 변경
- 상태판을 실제 기준으로 정상화했습니다.
- `P1-1`은 `[진행중]`, `P1-2`~`P1-4`는 `[완료]`, `P2-1`은 `[보류]`, `P2-2`~`P2-5`와 `P3-*`는 `[미착수]`로 정리했습니다.
- 전체 진행률은 `23% (3 / 13)`, Phase 1은 `75% (3 / 4)`, Phase 2는 `0% (0 / 5)`, Phase 3은 `0% (0 / 4)`로 다시 맞췄습니다.
- `dart-flow`는 DART 상세 화면 selector와 링크 문구 drift로 보이며, P1 header/IA 변경의 직접 영향으로 보지 않았습니다.
- `data-sources-settings`는 `Health API Aggregation`와 기대 문구 `health API 집계` 차이로 보이며, P1 IA 변경과 직접 연결되지 않습니다.
- `flow-planner-to-history`, `planning-quickstart-preview`는 `PlanningWorkspaceClient`의 test id와 상태 문구 drift로 분류했습니다.
- `news-settings-alert-rules`는 `뉴스 및 알림 설정`, `규칙 적용`, `알림함 ▶`, `Digest ▶` 등 현재 copy와 spec 기대값이 엇갈리는 문제로 분류했습니다.
- `smoke`의 recommend render 실패는 `recommend-root`, `recommend-submit` selector 부재와 `/api/sources/status` 500 흔적이 함께 보여 P1 직접 회귀보다 recommend surface 자체 후속 이슈로 분리했습니다.
- 최종적으로 `P1-1` 구현 자체는 남기되, closeout은 별도 e2e follow-up 없이 닫지 않기로 했습니다.
