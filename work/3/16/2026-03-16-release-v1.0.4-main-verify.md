## 변경 이유
- `main` 기준 `v1.0.4` 릴리즈를 진행하려면 `pnpm release:verify`가 통과해야 했는데, 이전 라운드에서 stale UI 기대값과 planning v2 interpretation evidence 계약 차이 때문에 최종 `vitest run`이 막혔습니다.
- 사용자 화면 문구와 테스트 기대값을 현재 UI와 다시 맞추고, standalone report에서 계산 근거 detail이 빠지지 않도록 planning v2 evidence adapter를 최소 수정으로 복구할 필요가 있었습니다.

## 핵심 변경
- `InterpretationGuide`와 planning v3 news 알림 화면에 남아 있던 영문 section label을 쉬운 한국어로 바꾸고, 관련 테스트 기대값을 현재 UI 기준으로 맞췄습니다.
- planning v2 interpretation VM이 raw `summaryEvidence`도 다시 받아들여 진단별 `evidenceDetail`을 붙일 수 있도록 복구했고, report interpretation adapter는 raw evidence map을 넘기도록 되돌렸습니다.
- stale UI copy 테스트들(`dart-search`, `homeQuickRulesStatus`, `reportDashboardOverrides`, `planning-v3-news-*`)을 현재 제품 문구와 guard 구조에 맞게 정리했습니다.
- `tests/e2e/planning-v2-fast.spec.ts`, `tests/e2e/planning-v2-full.spec.ts`의 현재 접근성/리포트 UI 기대값을 release gate 기준에 맞춰 유지했습니다.
- `pnpm release:verify`를 끝까지 다시 실행해 `planning:v2:complete`, `planning:v2:regress`, 전체 `vitest run`, `planning:ssot:check`까지 통과를 확인했습니다.
