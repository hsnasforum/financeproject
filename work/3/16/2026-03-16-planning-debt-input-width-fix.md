# 2026-03-16 플래닝 부채·목표 입력 폭 수정

## 변경 파일
- `src/components/PlanningWorkspaceClient.tsx`

## 사용 skill
- `planning-gate-selector`: `/planning` 사용자 경로에 걸린 컴포넌트 수정이라 `git diff --check`와 `pnpm build`를 최소 검증으로 선택하는 데 사용.
- `work-log-closeout`: 오늘 라운드의 실제 변경 파일, 실행한 검증, 남은 리스크를 `/work` 형식으로 정리하는 데 사용.

## 변경 이유
- 플래닝 화면에서 부채를 추가하면 `대출 잔액`, `금리`, `최소 상환액`, `남은 개월` 입력 칸이 `flex` 줄 안에서 폭을 제대로 줄이지 못해 서로 밀리거나 겹쳐 보였습니다.
- 같은 화면의 일반 목표 카드도 `목표 금액`, `현재 금액`, `목표 시점` 입력이 셀 폭을 끝까지 쓰지 못해 정렬이 어색했습니다.
- 공통 compact input 토큰은 유지한 채, 실제로 폭 문제가 보인 부채 리스트와 목표 카드만 국소적으로 고칠 필요가 있었습니다.

## 핵심 변경
- 부채 리스트 카드의 금액/금리 입력 래퍼에 `min-w-0`를 추가했습니다.
- `대출 잔액`, `금리`, `최소 상환액`, `남은 개월` 입력에 `flex-1 min-w-0`를 추가해 행 안에서 남은 폭을 꽉 쓰도록 맞췄습니다.
- 일반 목표 카드의 `목표 금액`, `현재 금액`, `목표 시점` 입력에는 `w-full`을 추가해 그리드 셀 폭을 끝까지 쓰도록 맞췄습니다.
- `bodyCompactFieldClassName` 공통 정의는 건드리지 않고, 실제 문제 구간인 `PlanningWorkspaceClient.tsx`의 부채/목표 입력만 최소 수정했습니다.
- 부채 ID, 이름, 목표 우선순위, 목표 최소 납입액 등 다른 필드나 계산 로직은 변경하지 않았습니다.

## 검증
- `git diff --check -- src/components/PlanningWorkspaceClient.tsx`
- `pnpm build`

## 남은 리스크
- 같은 패턴의 `flex + 단위 suffix` 입력이나 compact input이 다른 플래닝 섹션에도 있으면 유사한 폭 문제가 다시 보일 수 있습니다.
- 이번 수정은 실제로 문제가 확인된 부채/목표 입력에 한정되어 있어, 다른 입력 카드의 반응형 배치는 별도 점검이 필요할 수 있습니다.
