# 2026-03-11 manager-first 반복 루프 고정

## 변경 이유
- 사용자가 같은 방식의 멀티 에이전트 진행을 여러 턴에 걸쳐 계속 반복할 수 있게, 저장소 차원의 고정 규칙과 운영 문서가 필요했다.
- 이번 저장소는 `/work` 기록을 이어 받아 다음 라운드를 진행하는 경우가 많아, manager-first 루프를 명시적으로 남겨 두는 편이 안전하다.

## 이번 변경
1. 루트 `AGENTS.md` 에 `manager` 선행 분해, 병렬 가능한 에이전트만 배정, `/work` 기록 갱신까지 포함하는 반복 실행 루프를 추가했다.
2. `multi_agent.md` 에 시작 신호, 분해, 배치, 구현, 검증, 기록, 반복 조건으로 이어지는 저장소용 기본 루프를 추가했다.
3. `README.md` 문서 목록에 `multi_agent.md` 를 연결해 운영자가 바로 찾을 수 있게 했다.

## 검증
1. `AGENTS.md` 와 `multi_agent.md` 의 manager-first 절차 문구 확인
2. `README.md` 문서 링크 확인

## 검증 결과
- 루트 `AGENTS.md` 에 manager-first 반복 루프가 추가됐다.
- `multi_agent.md` 에 저장소용 반복 실행 절차가 추가됐다.
- `README.md` 에 `multi_agent.md` 링크가 추가됐다.

## 남은 리스크
- 이 문서는 반복 절차를 고정하지만, 각 라운드의 실제 우선순위 판단과 검증 범위 선택은 여전히 작업 내용에 따라 달라진다.
- 이후 더 구체적인 역할 프롬프트가 필요하면 `scripts/prompts/multi-agent/*.md` 까지 같은 기준으로 맞춰야 한다.
