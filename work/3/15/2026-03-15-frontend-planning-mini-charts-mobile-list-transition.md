# 2026-03-15 frontend planning-mini-charts-mobile-list-transition

## 목적
- `PlanningMiniCharts`의 상세 수치(Sampled Points) 영역을 모바일 환경에 최적화된 카드/리스트 구조로 전환
- 데스크톱에서는 정보 밀도가 높은 테이블 구조를 유지하고, 모바일에서는 가독성이 높은 카드 형태를 제공하여 반응형 사용자 경험 강화
- 프로젝트 디자인 가이드라인(모바일 테이블 사용 지양) 준수

## 변경 사항

### 1. PlanningMiniCharts 상세 수치 영역 반응형 최적화
- **데스크톱 (md 이상)**: 기존 `BodyTableFrame` 기반의 테이블 구조 유지. 헤더 대문자화 및 자간 조정 스타일 적용.
- **모바일 (md 미만)**: 테이블을 제거하고 개별 데이터 포인트(M1, M13 등)를 하나의 카드로 구성하는 리스트 구조 도입.
  - 각 카드는 상단에 시점(Index)을 표시하고, 하단에 순자산/현금/총부채를 3열 그리드로 배치하여 한눈에 비교 가능하게 설계.
  - 지표 성격에 맞는 색상(Emerald, Sky, Amber)을 수치에 적용하여 위계 강조.

### 2. Design Spec 반영
- **`docs/frontend-design-spec.md`**: `Mobile Data Display` 원칙을 명문화하여 모바일에서 테이블 대신 카드/리스트 사용을 권장하고, 데스크톱과의 정보 위계 일관성 유지 강조.

## 제외 사항
- **Sparkline Tooltip 미세조정**: 이전 라운드에서 이미 기능적으로 완료된 축이므로, 이번 모바일 리스트 전환 작업의 집중도를 위해 범위에서 제외함.

## 검증 결과
- **Lint**: `pnpm lint` 통과 (기존 legacy warning 2건 제외 clean).
- **Build**: `pnpm build` 통과.
- **UI/UX**: 모바일 가로 너비 부족으로 인한 테이블 깨짐 현상을 근본적으로 해결하고, 카드 한 장 안에서 핵심 수치 3개를 직관적으로 비교할 수 있는 구조임을 확인.

## 남은 UI debt
- 카드형 리스트 내 수치 클릭 시 해당 시점의 상세 데이터로 바로 이동하는 기능 (현재는 정적 노출).
- 모바일에서 데이터 포인트가 많아질 경우를 대비한 가로 스크롤 캐러셀 또는 더보기(Pagination) 검토.
