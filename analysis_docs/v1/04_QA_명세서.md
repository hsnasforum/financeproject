# 04. QA 명세서

## 1. 문서 목적

이 문서는 `financeproject`의 현재 저장소를 기준으로
회귀 테스트 범위, 보안 가드 검증, 상태 분리 기준, 자동화 우선순위를 정의합니다.

핵심 원칙
- 문서와 테스트를 분리하지 않습니다.
- 현재 존재하는 e2e 자산을 기준선으로 사용합니다.
- 기능 QA와 노출 정책 QA를 같은 수준으로 다룹니다.
- `Loading / Empty / Error / Disabled / Partial Success`를 모든 핵심 화면의 완료 기준에 포함합니다.

## 4. 테스트 환경 매트릭스

| 환경 | 목적 | 필수 여부 |
|---|---|---|
| Local dev | 모든 기능 및 local-only 경로 검증 | 필수 |
| Production-like non-local host | `/debug/*` 노출 정책 검증, `/dev/*` `/ops/*`는 [검증 필요] 수동 점검 | 필수 |
| Empty data | onboarding/빈 상태 확인 | 필수 |
| Seeded golden data | deterministic replay, report contract 확인 | 필수 |
| Snapshot missing/stale | assumptions 관련 예외 확인 | 필수 |
| Product DB empty | recommend fallback 확인 | 권장 |
| Partial failure induced | stage 단위 부분 실패 UI 확인 | 권장 |

## 5. 공통 완료 기준

각 화면/기능은 아래 항목이 모두 통과해야 완료로 간주합니다.

| 구분 | 기준 |
|---|---|
| Loading | skeleton/placeholder가 레이아웃을 크게 흔들지 않음 |
| Empty | 이유 설명과 기본 CTA 1개가 있음 |
| Error | 안전한 문구와 retry 또는 복구 경로가 있음 |
| Disabled | 비활성 이유가 사용자 근처에 보임 |
| Partial Success | 성공한 데이터는 유지하고 실패 단계만 명확히 표기 |
| Security | same-origin / CSRF / confirmText / local-only 정책 준수 |
| Traceability | run/report/recommend 결과를 다시 찾을 수 있음 |
