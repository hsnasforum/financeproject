# 05. 개선로드맵 / 백로그

## 1. 목적

이 백로그는 `financeproject`를 더 나은 방향으로 가져가기 위해
기술/기획/UI/QA 개선 항목을 우선순위 기준으로 정리한 실행용 목록입니다.

---

## 2. 우선순위 기준

| 우선순위 | 기준 |
|---|---|
| P0 | 제품 구조 이해도 또는 운영 안정성에 직접 영향 |
| P1 | 핵심 사용자 가치 상승 또는 유지보수 비용 절감 |
| P2 | 중장기 확장성 강화 |

---

## 3. 에픽 목록

## EPIC-01. 제품 경계 재정의

### BG-001 Public / Ops / Labs / Dev route policy 확정
- 우선순위: P0
- 문제: 현재는 문서상 분리 의식이 강하지만 사용자 IA에서는 혼재 가능성 존재
- 산출물
  - route classification 표준문서
  - 메뉴 노출 규칙
  - production 노출/비노출 체크리스트
- 완료 기준
  - `docs/current-screens.md`
  - 메뉴 구조
  - QA 명세서
  - e2e 노출 정책
  위 4개가 동일 기준을 사용

### BG-002 `/dashboard` 역할 재정의
- 우선순위: P0
- 옵션 A: 사용자 홈으로 축소
- 옵션 B: 사용자 홈 + 상태센터 이원화
- 완료 기준
  - 대시보드에 primary CTA 1개 축 명확
  - 운영 상태 섹션이 사용자 행동을 방해하지 않음

### BG-003 Planning v3 노출정책 확정
- 우선순위: P0
- 산출물
  - beta/labs 배지
  - 메뉴 비노출 규칙
  - deep link 허용 범위
- 완료 기준
  - v3 화면은 사용자가 “정식 기능”으로 오해하지 않음

## EPIC-03. 사용자 여정 강화

### BG-020 Planning → Report → Recommend 흐름 연결
- 우선순위: P1
- 개선안

  - report top actions에서 추천/실행 CTA 연결
  - recommendation history를 decision history로 재배치
- KPI

  - run → recommend 전환율

### BG-021 상품탐색과 추천 역할 분리
- 우선순위: P1
- 원칙

- 완료 기준

### BG-022 DART 검색/모니터 진입 문구 통일
- 우선순위: P0
- 완료 기준

## 6. 최종 요약

가장 먼저 손대야 할 것은 기능 추가가 아니라 아래 4개입니다.

1. 경로/메뉴/노출 정책 정리
2. canonical DTO 정리
3. planning과 recommend 연결
4. QA와 문서의 동기화

이 4개가 정리되면 이후 기능 개발 속도와 안전성이 동시에 올라갈 가능성이 높습니다.
