# Planning v2 Release Notes (1.0.4)

- Version: `1.0.4`
- Date: `2026-03-16`

## 변경 요약

### 1. 공식 public UI 밝은 톤 정리 마감
- 공식 사용자 화면의 밝은 톤 정리를 마감했습니다.
- 홈, 추천, 혜택, 기업 탐색, 알림 설정, 피드백, 공공 데이터, 플래닝 결과 화면의 잔여 dark-heavy 블록과 dark accent를 정리했습니다.
- 기본 베이스는 밝은 1안 톤으로 유지하고, dark tone은 badge, CTA, overlay처럼 제한된 포인트에만 남겼습니다.

### 2. 공식 사용자 화면 한글 copy 및 내부 값 노출 정리
- 공식 사용자 화면에 남아 있던 영문 라벨과 내부 구현값 노출을 한국어 중심으로 정리했습니다.
- `balanced`, `fetchedAt`, `Source`, `Score Breakdown`, `Min Score` 같은 값은 사용자용 한글 표현으로 바꿨습니다.
- 리포트·혜택·기업 탐색·홈 카드의 설명 문구도 비전문가 기준으로 더 쉽게 읽히도록 다듬었습니다.

### 3. `/planning/reports` 계산 근거 중앙 오버레이 전환
- `/planning/reports` 상품 후보 비교표의 계산 근거 보기를 셀 내부 확장에서 가운데 오버레이로 전환했습니다.
- 비교표를 읽던 흐름을 유지한 채 `공식`, `입력값`, `가정`을 따로 확인할 수 있게 했습니다.
- 비교표 상단과 계산 가정 블록의 영문 라벨도 함께 한글화했습니다.

### 4. `/planning` 부채·목표 입력 폭 정렬 수정
- `/planning` 부채 추가 카드와 목표 카드에서 입력 칸 폭이 줄지 않아 겹쳐 보이던 문제를 수정했습니다.
- `대출 잔액`, `금리`, `최소 상환액`, `남은 개월`, `목표 금액`, `현재 금액`, `목표 시점` 입력이 셀 폭을 정상적으로 쓰도록 맞췄습니다.

### 5. `analysis_docs` 정적 분석 문서 세트 정비
- `analysis_docs/**` 정적 분석 문서 세트를 현행 코드 기준으로 재검증했습니다.
- `02_화면정의서`, `03_DTO_API_명세서`, `04_QA_명세서`를 보정했고, 남아 있던 data-sources/CSRF 설명도 확인 가능한 범위까지 해소했습니다.
- 운영 메모에는 Git 추적 권장안과 `/work` 대표본 규칙 제안을 정리했습니다.

## 사용자 영향
- 공식 사용자 화면의 용어와 안내 문구가 더 쉽게 읽히고, 내부 구현값이 직접 노출되는 일이 줄었습니다.
- 플래닝 리포트의 계산 근거 확인 흐름이 표 레이아웃을 깨지 않도록 개선됐습니다.
- 플래닝 입력 카드의 반응형 정렬이 안정돼 부채·목표 입력이 덜 불편해졌습니다.

## 기술/운영 메모
- 이번 버전은 계산 엔진이나 API 계약을 크게 바꾸는 릴리즈가 아니라, 사용자 경험 정리와 문서 정비 중심의 후속 릴리즈입니다.
- `analysis_docs/**`는 기준 문서 세트로 정비됐고, 실제 Git 추적 정책과 `/work` 운영 규칙은 별도 운영 결정으로 이어질 수 있습니다.
