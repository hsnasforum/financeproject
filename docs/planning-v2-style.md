# Planning v2 Style Guide (User-facing)

## Tone Rules
- 권유/단정 표현 금지: `추천합니다`, `무조건`, `정답` 같은 문구를 쓰지 않습니다.
- 확률 문장은 항상 `모델 기반 결과이며 보장이 아님` 맥락을 포함합니다.
- 금액은 원 단위로 표시하고, 해석은 `부족액/여유` 중심으로 설명합니다.
- 위험 표현은 아래 고정 문장을 우선 사용합니다.

## Severity Wording
- `risk`: `계획이 쉽게 흔들릴 수 있습니다`
- `warn`: `안전마진이 부족할 수 있습니다`
- `ok`: `큰 위험 신호는 크지 않습니다`

## Visibility Rules
- 경고 코드명(`CONTRIBUTION_SKIPPED` 등)은 기본 화면에서 숨깁니다.
- 코드명은 고급 보기에서만 노출합니다.
