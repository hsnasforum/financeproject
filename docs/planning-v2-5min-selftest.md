# Planning v2 5분 셀프 테스트

목표: 처음 사용하는 사용자도 5분 안에 `/planning` 흐름을 완주할 수 있는지 확인합니다.

## 사전 준비
- 개발 서버 실행 상태 확인
- 필요 시 초기 데이터 준비: `pnpm planning:v2:seed`

## 사용자 체크리스트
- [ ] `/planning`에서 샘플 프로필을 불러오거나 새 프로필을 생성했다.
- [ ] `계획 실행`을 눌러 Run을 실행했다(시나리오 포함).
- [ ] 결과 상단 `해석 가이드`에서 `위험/주의/양호` 상태가 표시된다.
- [ ] `Warnings Summary`가 코드 덤프가 아니라 사람이 읽을 수 있는 문장으로 보인다.
- [ ] `실행 저장` 버튼으로 run 저장에 성공했다.
- [ ] `/planning/runs`에서 방금 저장한 run을 확인했다.
- [ ] run 상세에서 HTML 리포트 보기(또는 다운로드)에 성공했다.
- [ ] (선택) snapshot history를 선택해 실행에 성공했다.

## 점검용 명령
- `pnpm planning:v2:seed`
- `pnpm planning:v2:complete`
- 서버 실행 후: `pnpm planning:v2:acceptance`

## 판정 기준
- 위 체크리스트가 모두 완료되면 사용자 관점 기본 흐름은 통과입니다.
- 1개라도 실패하면 실패한 단계의 화면 문구/레이블/버튼 상태를 우선 수정합니다.
