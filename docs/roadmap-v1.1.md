# v1.1 Roadmap

## 우선순위

1. [P0] 운영 자동복구 고도화
- FIX/CHAIN 실패 패턴 자동 분류를 강화하고, 실패 재발 시 단계별 재시도 정책을 도입한다.

2. [P0] Ops Hub 통합 실행 안정화
- /ops에서 doctor, backup, cleanup, audit 액션의 실행 결과를 공통 형식으로 표시하고 실패 원인 추적을 단순화한다.

3. [P1] 정책 기반 운영 가드 확장
- retention 외에도 실행 시간대/빈도/최대 실행 횟수 제한 정책을 도입해 오작동 위험을 줄인다.

4. [P1] 감사 로그 검색/필터링
- event/route/기간/결과 기준 필터와 CSV export를 추가해 운영 감사 효율을 높인다.

5. [P1] 운영 KPI 대시보드
- 복구 성공률, 평균 복구 시간(MTTR), 실패 재발률, 자동 티켓 누적량을 시계열로 시각화한다.

6. [P2] 시뮬레이션/드라이런 리포트 강화
- dryRun 결과에 예상 변경량/영향 범위/rollback 난이도 지표를 포함한다.

7. [P2] 운영 문서 자동 동기화
- runbook/정책/감사로그 변경 시 docs를 자동 갱신하고 릴리즈 노트 초안을 생성한다.
